AMP和MIP简介

发表于 2018-03-23

google先出了个AMP - Accelerated Mobile Pages(加速移动页面),百度仿照出了个MIP - Mobile Instant Pages- 移动网页加速器。那么这2个到底是干嘛的呢？

其实这2个东西的用处就是，你做个网页，使用google/百度制定的一些规则，你的页面加载速度会更快，据说有15%~85%的性能提升，另外还有一个隐藏功能没有大肆宣扬，那就是可以提升搜索引擎的排名(也就是seo)。

我们都知道，排名是和钱挂钩的，因为这东西还是有学习的必要的。

ps:由于国内墙的关系，故例子都举MIP的

采用内联样式

出于速度考虑，建议內联使用 CSS 样式。所有样式写在 <style mip-custom></style>中，并且style标签仅允许出现一次

替换禁用的标签

在使用移动页面加速技术的时候，很多标签是不能使用的。例如<img>、<video>、<iframe>等都是不能使用的，需要使用其他标签替换或者直接禁止使用（如<iframe>），例如在MIP中，<img>标签需要替换成<mip-img>。

替换和禁用列表

使用内置的组件

不允许出现自定义js代码，如果一定要使用js，则必须使用MIP内置的组件

内置的组件

目前组件分为内置组件(可直接使用的，无需引入额外的js)和个性化组件(需引入额外的组件)。

一些弊端

原有代码改动大

不支持与其他框架结合使用，如react,angular,vue。

需要学习的量非常多。

结论

如果你的页面有涉及到SEO，那不用怀疑，推荐使用MIP。

如果你的页面想提高加载速度，那可以根据投入产出比来考虑是否使用，因为要提高加载速度还有很多其他的方法，性价比还更高。
